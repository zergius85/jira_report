### 3. 🔴 Risk Zone — отладка и логирование

**Проблема:**
- Risk Zone всегда показывал пустой результат
- Не было понятно, почему задачи не попадают в отчёт

**Решение:**
- Добавлено подробное логирование процесса проверки
- Логирование количества задач на входе
- Логирование количества найденных рисковых задач
- Предупреждение если `issues_normal` не определён или пуст

**Диагностика:**
```python
logger.info(f"🔍 Проверка Risk Zone... issues_normal: {'issues_normal' in locals()}")
logger.info(f"   Найдено {len(issues_normal)} задач для проверки")
logger.info(f"   Найдено {len(risk_issues)} рисковых задач")
```

**Факторы риска (проверяются):**
1. **Без исполнителя** — `assignee = null`
2. **Просрочена** — `duedate < today AND status NOT IN (Closed, Done)`
3. **Не двигается** — `updated < today - 5 дней AND status NOT IN (Closed, Done)`

**Порог неактивности:**
- Настраивается в `core/config.py`
- По умолчанию: `RISK_ZONE_INACTIVITY_THRESHOLD = 5` (дней)

## 🐛 Известные ограничения

### Risk Zone может быть пустым по причинам:

1. **Все задачи закрыты** — Risk Zone показывает только открытые задачи
2. **Нет задач без исполнителя** — все задачи назначены
3. **Нет просроченных задач** — все задачи в срок
4. **Все задачи обновлялись недавно** — менее 5 дней назад
5. **`issues_normal` пуст** — возможно, неправильный период отчёта

**Для диагностики:**
```bash
# Включите DEBUG логирование в .env
LOG_LEVEL=DEBUG

# Перезапустите службу
sudo systemctl restart jira-report-dev

# Смотрите логи
journalctl -u jira-report-dev -f | grep "Risk Zone"
```

**Ожидаемые логи:**
```
🔍 Проверка Risk Zone... issues_normal: True
   Найдено 142 задач для проверки
   Найдено 5 рисковых задач
✅ Risk Zone заполнен: 5 задач
```

Или:
```
🔍 Проверка Risk Zone... issues_normal: True
   Найдено 50 задач для проверки
   Найдено 0 рисковых задач
ℹ️  Risk Zone пуст (нет рисковых задач)
```
