## Обзор блоков

| Блок | Описание | Стандартный режим | Extra verbose |
|------|----------|-------------------|---------------|
| **summary** | Сводка по проектам | 7 колонок | 7 колонок (+[id] в полях) |
| **assignees** | Нагрузка по исполнителям | 7 колонок | 7 колонок (+[id] в полях) |
| **detail** | Детализация по задачам | 10 колонок | 10 колонок (+[id] в полях) |
| **issues** | Проблемные задачи | 8 колонок | 8 колонок (+[id] в полях) |
| **internal** | Внутренние проекты | 10 колонок | 10 колонок (+[id] в полях) |
| **risk_zone** | Рисковые задачи | 7 колонок | 7 колонок (+[id] в полях) |

**Важно:** В extra_verbose режиме **не добавляется отдельная колонка ID**. Вместо этого ID добавляется в квадратных скобках к значениям полей.

## Сводка по проектам (summary)

Агрегированные данные по каждому проекту.

### Стандартный режим

| Колонка | Пример |
|---------|--------|
| Клиент (Проект) | `WEB` |
| Задач закрыто | `15` |
| Корректных | `12` |
| С ошибками | `3` |
| Оценка (ч) | `45.5` |
| Факт (ч) | `42.0` |
| Отклонение | `3.5` |

### Extra verbose режим

| Колонка | Пример | Tooltip |
|---------|--------|---------|
| Клиент (Проект) | `WEB [10001]` | `project = '10001'` |
| Задач закрыто | `15` | — |
| Корректных | `12` | — |
| С ошибками | `3` | — |
| Оценка (ч) | `45.5 [timeoriginalestimate]` | `timeoriginalestimate` |
| Факт (ч) | `42.0 [timespent]` | `timespent` |
| Отклонение | `3.5` | — |

## Детализация по задачам (detail)

Полные данные по каждой закрытой задаче.

### Стандартный режим (10 колонок)

| Колонка | Пример |
|---------|--------|
| URL | `https://jira/browse/WEB-123` |
| Дата решения | `2026-03-10` |
| Дата исполнения | `2026-03-10` |
| Дата создания | `2026-03-01` |
| Проект | `WEB` |
| Статус | `Done (done)` |
| Задача | `Исправить баг` |
| Исполнитель | `Иванов Иван` |
| Факт (ч) | `2.5` |
| Тип | `Bug` |

### Extra verbose режим (10 колонок)

| Колонка | Пример | Tooltip |
|---------|--------|---------|
| URL | `https://jira/browse/WEB-123 🔍` | `id = '10500'` |
| Дата решения | `2026-03-10 [resolutiondate]` | `resolutiondate` |
| Дата исполнения | `2026-03-10 [duedate]` | `duedate` |
| Дата создания | `2026-03-01 [created]` | `created` |
| Проект | `WEB [10001]` | `project = '10001'` |
| Статус | `Done (done) [5]` | `status = '5'` |
| Задача | `Исправить баг` | — |
| Исполнитель | `Иванов Иван [account-123]` | `assignee = 'account-123'` |
| Факт (ч) | `2.5 [timespent]` | `timespent` |
| Тип | `Bug [1]` | `issuetype = '1'` |

**Примечание:** Столбец URL сохраняет структуру как в issues (с иконкой 🔍 для отладки).

## Сравнение режимов

### Формат данных

| Поле | Стандартный | Extra verbose | Tooltip |
|------|-------------|---------------|---------|
| Проект | `WEB` | `WEB [10001]` | `project = '10001'` |
| Исполнитель | `Иванов Иван` | `Иванов Иван [account-123]` | `assignee = 'account-123'` |
| Статус | `Done (done)` | `Done (done) [5]` | `status = '5'` |
| Тип задачи | `Bug` | `Bug [1]` | `issuetype = '1'` |
| Дата | `2026-03-10` | `2026-03-10 [duedate]` | `duedate` |
| Факт (ч) | `2.5` | `2.5 [timespent]` | `timespent` |
| Автор | `Петров Петр` | `Петров Петр [account-456]` | `creator = 'account-456'` |
| URL | `https://...` | `https://... 🔍` | `id = '10500'` |

### Обозначения

| Обозначение | Описание |
|-------------|----------|
| `[10001]` | ID объекта (project, issue, status и т.д.) |
| `[timespent]` | Название поля Jira (field name) |
| `🔍` | Иконка отладки для URL |

### Количество колонок

| Блок | Стандартный | Extra verbose | Изменение |
|------|-------------|---------------|-----------|
| summary | 7 | 7 | 0 |
| assignees | 7 | 7 | 0 |
| detail | 10 | 10 | 0 |
| issues | 8 | 8 | 0 |
| internal | 10 | 10 | 0 |
| risk_zone | 7 | 7 | 0 |

**Важно:** Количество колонок **не меняется** в extra_verbose режиме. ID добавляется в квадратных скобках к значениям полей.

## Всплывающие подсказки (Tooltip)

В extra_verbose режиме при наведении на элемент в квадратных скобках отображается информация:

| Элемент | Tooltip формат |
|---------|----------------|
| Проект `[10001]` | `project = '10001'` |
| Исполнитель `[account-123]` | `assignee = 'account-123'` |
| Статус `[5]` | `status = '5'` |
| Тип задачи `[1]` | `issuetype = '1'` |
| Приоритет `[3]` | `priority = '3'` |
| Дата `[duedate]` | `duedate` (название поля) |
| Факт (ч) `[timespent]` | `timespent` (название поля) |
| URL `🔍` | `id = '10500'` (ID задачи) |

## Примеры использования

### Консольный режим

```bash
# Стандартный режим
python3 jira_report.py -b summary,detail

# Extra verbose режим
python3 jira_report.py -b summary,detail -vv
```

### Web-интерфейс

Чекбокс "Показывать ID объектов (extra verbose)" в форме отчёта.

### API

```json
{
  "projects": ["WEB"],
  "blocks": ["summary", "detail"],
  "extra_verbose": true
}
```
