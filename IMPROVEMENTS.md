## Март 2026 — Оптимизация получения проектов

**Статус:** ✅ Завершено (11 марта 2026)

### Проблема
Метод `jira.projects()` делал много запросов к API (по одному на проект) и возвращал все проекты включая закрытые. Проверка флага `archived` не всегда работала в Jira Server/Data Center.

**Количество запросов:** N+1 (где N = количество проектов)
- Для 50 проектов: 51 запрос
- Для 100 проектов: 101 запрос

### Решение

#### Новый подход: один JQL запрос вместо N
Вместо `jira.projects()` используется `search_issues()` с JQL запросом для получения всех задач за период. Проекты извлекаются из возвращённых задач.

**JQL запрос:**
```jql
created >= '2026-03-06' AND created <= '2026-05-08'
ORDER BY created DESC
```

#### Преимущества
1. **Экономия запросов:** 1 запрос вместо N+1 (~98% экономия)
2. **Автоматическая фильтрация:** Закрытые проекты не возвращаются (в них нет новых задач)
3. **Исключение проектов:** `EXCLUDED_PROJECTS` применяется корректно
4. **Совместимость:** Работает с Jira Server/DC без флага `archived`

### Изменения

**core/jira_report.py:**
```python
# До:
all_projects = jira.projects()  # Много запросов
for proj in all_projects:
    if proj.archived:  # Не всегда работает
        continue

# После:
jql_all_projects = f"created >= '{start_date}' AND created <= '{end_date}'"
all_issues_temp = search_all_issues(jira, jql_all_projects, fields='project')

# Извлекаем уникальные проекты из задач
projects_map = {}
for issue in all_issues_temp:
    project = issue.fields.project
    if project.key not in EXCLUDED_PROJECTS:
        projects_map[project.key] = project.name
```

### Метрики

| Метрика | До | После | Экономия |
|---------|-----|-------|----------|
| Запросов к Jira | N+1 | 1 | ~98% |
| Для 50 проектов | 51 | 1 | 50 запросов |
| Для 100 проектов | 101 | 1 | 100 запросов |
| Закрытые проекты | Возвращались | Исключены | 100% |

## Март 2026 — Исправление extra_verbose режима

**Статус:** ✅ Завершено (11 марта 2026)

### Проблема
При включении режима extra_verbose в web-интерфейсе многие столбцы становились пустыми. Заголовки столбцов менялись (добавлялись `[id]`, `[duedate]`, `[project]` и т.д.), но данные в ячейках не соответствовали новым заголовкам.

**Причина:** Функция `get_column_order()` возвращала изменённые заголовки, но генераторы данных не формировали значения в соответствующем формате.

### Решение

#### 1. Заголовки столбцов больше не меняются
- При включенном extra_verbose **добавляется только отдельная колонка 'ID'** после 'URL'
- Все остальные заголовки остаются неизменными

#### 2. Данные в ячейках содержат ID и title
- **Формат значения:** `value [id]` или `value [field_name]`
- **Всплывающая подсказка (title):** `field_name = 'id'`

**Пример:**
```
Проект: "WEB [10001]" (title: "project = '10001'")
Исполнитель: "Иванов [account-123]" (title: "assignee = 'account-123'")
Факт (ч): "1.5 [timespent]" (title: "timespent")
```

#### 3. JavaScript обработчики
- `extractCleanValue()` — извлекает чистое значение без `[id]`
- `extractTitle()` — извлекает title из формата `value [field_name]`
- `formatCellWithTooltip()` — рендерит ячейки с tooltip

### Изменённые файлы

| Файл | Изменения |
|------|-----------|
| `core/jira_report.py` | `get_column_order()` — заголовки не меняются, только добавляется колонка 'ID' |
| `core/report_generator.py` | `IssueDataExtractor`, `ReportBlockGenerator`, `_fetch_internal_projects` — форматирование данных с ID |
| `templates/index.html` | `renderTable()`, `renderIssuesTable()` — извлечение чистых значений и title |
| `tests/test_report_generator.py` | Обновлены тесты для новых форматов данных |

### Тесты
Все 93 теста проходят ✅
