## 🔧 Технические долги (Март 2026)

По результатам аудита кода от 9 марта 2026 выявлен ряд проблем, требующих исправления.

### P0 — Критические проблемы

| № | Проблема | Описание | Статус |
|---|----------|----------|--------|
| **P0-1** | Обработка исключений в API | Отсутствие try-except в `api_report()`, `api_download()` | ⏳ Запланировано |
| **P0-2** | Rate limiting | Нет ограничения частоты запросов к API | ⏳ Запланировано |
| **P0-3** | Утечка памяти в кэше | `_project_cache` растёт бесконечно | ⏳ Запланировано |
| **P0-4** | SSL_VERIFY глобально отключается | `os.environ['REQUESTS_CA_BUNDLE'] = ''` влияет на все запросы | ⏳ Запланировано |

### P1 — Средний приоритет

| № | Проблема | Описание | Статус |
|---|----------|----------|--------|
| **P1-1** | Нарушение SRP | `jira_report.py` — 1255 строк, `web/app.py` — 1313 строк | ⏳ Запланировано |
| **P1-2** | Дублирование кода | Логика в `jira_report.py` и `report_generator.py` дублируется | ⏳ Запланировано |
| **P1-3** | Отсутствие типизации | Массовое использование `Any` вместо конкретных типов | ⏳ Запланировано |
| **P1-4** | N+1 запросов к Jira | Для N проектов делается 2N запросов | ⏳ Запланировано |

### P2 — Низкий приоритет

| № | Проблема | Описание | Статус |
|---|----------|----------|--------|
| **P2-1** | Хардкод значений | Магические числа в коде (50, 5, 365) | ⏳ Запланировано |
| **P2-2** | Покрытие тестами | Нет тестов для `ReportGenerator`, `api_report()` | ⏳ Запланировано |
| **P2-3** | Timezone хардкод | `timezone='Europe/Moscow'` без конфигурации | ⏳ Запланировано |
| **P2-4** | Transaction management | Отсутствие proper transaction handling в БД | ⏳ Запланировано |
| **P2-5** | Кэш только в production | В dev-режиме кэш отключён | ⏳ Запланировано |
